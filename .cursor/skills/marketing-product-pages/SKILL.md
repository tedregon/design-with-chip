# Marketing Product Pages

## Purpose
Use this skill to produce high-converting product pages that are clear, scannable, and aligned with the quality bar set by the Acting Coach page.

## When To Use
- User asks to write or improve a product page, landing page, or sales page.
- User asks to improve conversion, clarity, positioning, or CTA quality.
- User asks to update copy for WordJam, Vigor, or similar product sections.

## Inputs To Gather First
- Product name and one-line promise.
- Target audience and top pain point.
- Core outcomes the product delivers.
- Distinct mechanism (how it works, why it is different).
- Proof available (results, testimonials, screenshots, specifics).
- Primary call to action and desired user action.

If critical details are missing, ask for them briefly before rewriting.

## Messaging Framework
Apply this structure from top to bottom:

1. Hero
- Who this is for.
- Main outcome in plain language.
- One focused CTA.

2. Problem
- Name the pain in the user's words.
- Show the cost of not solving it.

3. Solution
- Explain the product's mechanism, not just features.
- Tie each capability to a user outcome.

4. Proof
- Add concrete evidence: examples, numbers, testimonials, or believable specifics.
- Reduce risk with credibility signals.

5. CTA
- Use one primary action phrase repeated consistently.
- Make the next step obvious and low-friction.

## Copy Rules
- Lead with outcomes before features.
- Prefer specific claims over vague adjectives.
- Keep headers short and concrete.
- Remove filler, repeated ideas, and generic hype.
- Use short paragraphs and skimmable sections.
- Keep terminology consistent across the page.

## Rewrite Workflow
1. Audit the current page quickly.
2. Identify the biggest messaging gaps (top 3).
3. Propose a clean section outline.
4. Rewrite section-by-section in priority order:
   - Hero
   - Problem/Solution
   - Proof
   - CTA blocks
5. Tighten wording for clarity and momentum.
6. Run final QA checklist.

## Required Output Format
Always return results in this order:

1. Quick Audit
- 3-6 bullets on current strengths and weaknesses.

2. Proposed Structure
- Section list with one sentence intent per section.

3. Revised Copy
- New copy grouped by section with clear labels.

4. Variants
- 3 headline options.
- 2 CTA options.

5. QA Pass
- Confirm clarity, consistency, and CTA alignment.

## Quality Bar
Final copy should be:
- Clear in under 10 seconds of scanning.
- Specific enough to feel credible.
- Focused on user outcomes, not internal product language.
- Actionable with one obvious next step.
